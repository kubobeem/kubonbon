GitHub Pages rewards simple architecture. Relative links and static assets resolve reliably when everything is rooted in the repository publish folder.

Checklist before push

  • Confirm every navigation link is relative.
  • Ensure data/posts.json is valid JSON.
  • Verify each post entry points to an existing file path.
  • Use a 404.html page to keep error states intentional.

After publishing

Open the GitHub Pages URL, click through home, detail, and tags pages, then test one intentionally invalid route to confirm 404 behavior.
