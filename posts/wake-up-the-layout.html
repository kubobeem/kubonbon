A strong blog foundation is mostly structural discipline. Define the visual language once, then keep every new post inside the same content contract.

Start with reusable sections

Build cards, hero blocks, metadata lines, and tag chips as reusable primitives. Once these are stable, every post ships faster and with fewer regressions.

Keep content and layout separate

This project reads post metadata from a single manifest. New articles are added as files, then connected with one new entry in data/posts.json.

That separation keeps your publishing loop clean: write, link, push, and verify.
